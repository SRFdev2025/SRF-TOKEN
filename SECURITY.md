# Security Policy

## Reporting a Vulnerability
If you discover any security vulnerability in the SRF Token smart contract or any related systems, please report it responsibly via email:
dev@srftoken.com

We are committed to reviewing and fixing any vulnerabilities as quickly as possible.

## Smart Contract Audits
The SRF Token contract is planned for a full external audit.  
Audit provider (TBA): Under negotiation.

## Security Best Practices
- Contract follows standard ERC20 patterns.
- No mint function after deployment.
- Ownership secured via multi-sig (planned).
- Liquidity lock will be published after listing.
- Bug bounty program planned after the audit.
